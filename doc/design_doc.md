# 多精度 FMA 单元设计文档

## 1. 概述

本文档描述了一个多精度 Fused Multiply-Add（FMA）运算单元的设计，支持 FP32、FP16、FP8（E4M3）和 INT8 四种数据格式。设计采用分阶段渐进实现策略，核心思想是**最大化运算器复用**——乘法器、移位器、CSA 压缩器和加法器跨精度共享，通过 `mode[2:0]` 控制信号切换位宽与数据格式。

### 1.1 支持的操作模式

| `mode[2:0]` | 操作模式 | 说明 |
|-------------|---------|------|
| `3'b000` | FP32 FMA | `result = A×B + C`（单精度浮点） |
| `3'b001` | FP16 FMA | `result = A×B + C`（半精度浮点） |
| `3'b010` | FP8×4 FMA | 四路并行 FP8 FMA |
| `3'b011` | INT8 MAD | `result = A_int×B_int + C_int`（8位整数乘加） |
| `3'b100` | FP16→FP32 Acc | FP16×FP16 累加到 FP32 |
| `3'b101` | FP8×4→FP32 Acc | 四路 FP8×FP8 累加到 FP32 |

### 1.2 设计约束

- **运算器复用优先**：乘法器、移位器、加法器、CSA 树跨精度共享
- **流水线对齐**：所有模式与 3 级流水线对齐（Stage1=乘法，Stage2=对阶/CSA，Stage3=规格化/舍入）
- **格式解析前端**：统一解析 FP32/FP16/FP8/INT8，提取符号位、指数、尾数
- **Bias 修正统一处理**：各精度 bias（127/15/7）在指数加法阶段统一修正

## 2. 顶层模块接口

### 模块名：`fma_top`

```verilog
module fma_top (
    input  wire        clk,
    input  wire        rst_n,
    input  wire [2:0]  mode,       // 操作模式
    input  wire        valid_in,   // 输入有效
    input  wire [31:0] A,          // 操作数 A
    input  wire [31:0] B,          // 操作数 B
    input  wire [31:0] C,          // 操作数 C / 累加初值
    input  wire        acc_en,     // 累加使能（阶段3）
    input  wire [31:0] acc_in,     // 外部累加器初值（阶段3）
    output wire [31:0] result,     // 计算结果
    output wire [31:0] acc_out,    // 累加结果（阶段3）
    output wire        valid_out,  // 输出有效
    output wire        overflow,   // 溢出标志
    output wire        underflow,  // 下溢标志
    output wire        inexact     // 不精确标志
);
```

### 数据打包约定

| 模式 | A[31:0] | B[31:0] | C[31:0] | result[31:0] |
|------|---------|---------|---------|--------------|
| FP32 | FP32 数据 | FP32 数据 | FP32 数据 | FP32 结果 |
| FP16 | `[15:0]`=FP16 | `[15:0]`=FP16 | `[15:0]`=FP16 | `[15:0]`=FP16 结果 |
| FP8×4 | 4路 FP8 打包 | 4路 FP8 打包 | 4路 FP8 打包 | 4路 FP8 结果打包 |
| INT8 | `[7:0]`=INT8 | `[7:0]`=INT8 | `[15:0]`=INT16 | `[15:0]`=INT16 结果 |
| FP16→FP32 Acc | `[15:0]`=FP16 | `[15:0]`=FP16 | 不使用 | FP32 累加结果 |
| FP8×4→FP32 Acc | 4路 FP8 | 4路 FP8 | 不使用 | FP32 累加结果 |

## 3. 阶段 0：基础 FP32 FMA

### 3.1 功能描述

$$result_{[31:0]} = A_{[31:0]} \times B_{[31:0]} + C_{[31:0]}$$

IEEE 754 单精度浮点格式：

| 字段 | 位域 | 宽度 |
|------|------|------|
| 符号位 | `[31]` | 1 bit |
| 指数位 | `[30:23]` | 8 bit（bias=127） |
| 尾数位 | `[22:0]` | 23 bit |

### 3.2 三级流水线数据通路

#### Pipeline Stage 1：乘法

- **符号计算**：`sign_product = A[31] ^ B[31]`
- **指数相加**：`exp_sum[8:0] = A[30:23] + B[30:23] - 8'd127`（9bit 防溢出）
- **尾数乘法**：`{1'b1, A[22:0]} × {1'b1, B[22:0]}` → `product[47:0]`（48bit 乘法器）

#### Pipeline Stage 2：对阶与 CSA 压缩

- **对阶偏移量**：`shift_amt = exp_sum - {1'b0, C[30:23]}`
- **C 尾数对齐**：通过 50bit 移位器将 `{1'b1, C[22:0]}` 移位对齐
- **CSA 压缩**：3-2 CSA 将乘积高位与 C 对齐尾数压缩为：
  - `csa_sum[49:0]`
  - `csa_carry[49:0]`

#### Pipeline Stage 3：加法、规格化与舍入

- **加法**：`final_sum[50:0] = csa_sum + csa_carry`
- **前导零检测（LZD）**：检测 `final_sum` 的前导零数量
- **规格化**：左移消除前导零，调整指数
- **舍入**：Round to Nearest Even（IEEE 754 默认舍入模式）
- **结果组装**：`result[31:0] = {sign, exp[7:0], mantissa[22:0]}`

### 3.3 关键运算器清单

| 运算器 | 位宽 | 数量 | 用途 |
|--------|------|------|------|
| 乘法器 | 48bit (24×24) | 1 | 尾数乘法 |
| 移位器 | 50bit | 1 | C 尾数对阶移位 |
| 3-2 CSA | 50bit | 1 | 乘积与 C 压缩 |
| 加法器 | 50bit | 1 | CSA 结果求和 |
| 规格化器/LZD | — | 1 | 前导零检测与规格化 |

### 3.4 特殊值处理

| 条件 | 结果 |
|------|------|
| A 或 B 为 NaN | 输出 NaN（`32'h7FC00000`） |
| A 或 B 为 Inf，另一个为零 | 输出 NaN |
| A 或 B 为 Inf，另一个非零 | 输出 Inf（符号由乘积决定） |
| A×B=Inf，C=Inf（异号） | 输出 NaN |
| C 为 NaN | 输出 NaN |
| A 或 B 为零 | 输出 C |
| C 为零 | 输出 A×B |
| 结果下溢 | 输出 denormal 或零，置 `underflow` |
| 结果上溢 | 输出 Inf，置 `overflow` |

## 4. 阶段 1：支持 FP16（两路）

### 4.1 功能描述

$$result_{[15:0]} = A_{[15:0]} \times B_{[15:0]} + C_{[15:0]}$$

IEEE 754 半精度浮点格式：

| 字段 | 位域 | 宽度 |
|------|------|------|
| 符号位 | `[15]` | 1 bit |
| 指数位 | `[14:10]` | 5 bit（bias=15） |
| 尾数位 | `[9:0]` | 10 bit |

### 4.2 模块接口变更

- 新增：`mode[1:0]`（`00`=FP32，`01`=FP16）
- FP16 输入使用 `A[15:0]`、`B[15:0]`、`C[15:0]`
- FP16 输出在 `result[15:0]`

### 4.3 运算器复用说明

#### 乘法器复用

- FP16 尾数乘法：`{1'b1, A[9:0]} × {1'b1, B[9:0]}` → 22bit 乘积
- **映射方式**：使用 FP32 48bit 乘法器的**低 22 位**，高位强制置 0
- 乘法器输入 MUX：
  - `mode==FP32`：`mul_a = {1'b1, A[22:0]}`，`mul_b = {1'b1, B[22:0]}`
  - `mode==FP16`：`mul_a = {13'b0, 1'b1, A[9:0]}`，`mul_b = {13'b0, 1'b1, B[9:0]}`

#### 指数加法器复用

- FP16 指数计算：`exp_sum[5:0] = A[14:10] + B[14:10] - 5'd15`
- 映射到 FP32 指数通路：`exp_fp32[7:0] = fp16_exp_sum[4:0] + 8'd112`

#### 移位器、CSA、加法器

- 与阶段 0 完全共享，`mode` 信号控制有效位宽

### 4.4 FP16 → FP32 中间格式扩展

| 字段 | 扩展规则 |
|------|---------|
| 尾数 | `product_fp32[22:0] = {fp16_product[21:0], 1'b0}` |
| 指数 | `exp_fp32[7:0] = fp16_exp_sum[4:0] + 8'd112` |

### 4.5 FP16 特殊值处理

- NaN：指数全 1（`5'h1F`），尾数非 0 → 输出 `16'h7E00`
- Inf：指数全 1（`5'h1F`），尾数全 0 → 输出对应符号的 Inf
- 零/denormal：与 FP32 处理逻辑一致，位宽缩减

## 5. 阶段 2：支持 FP8（四路并行）

### 5.1 功能描述

$$result^{(i)}_{[7:0]} = A^{(i)}_{[7:0]} \times B^{(i)}_{[7:0]} + C^{(i)}_{[7:0]}, \quad i \in \{0,1,2,3\}$$

FP8 E4M3 格式：

| 字段 | 位域 | 宽度 |
|------|------|------|
| 符号位 | `[7]` | 1 bit |
| 指数位 | `[6:3]` | 4 bit（bias=7） |
| 尾数位 | `[2:0]` | 3 bit |

### 5.2 模块接口变更

- `mode[1:0]` 扩展：`10`=FP8×4
- 输入打包：`A[7:0]`=Lane0, `A[15:8]`=Lane1, `A[23:16]`=Lane2, `A[31:24]`=Lane3
- 输出打包：`result[7:0]`=Lane0, ..., `result[31:24]`=Lane3

### 5.3 四路 FP8 数据通路

#### 解包

```
A_fp8[0] = A[7:0],    B_fp8[0] = B[7:0],    C_fp8[0] = C[7:0]
A_fp8[1] = A[15:8],   B_fp8[1] = B[15:8],   C_fp8[1] = C[15:8]
A_fp8[2] = A[23:16],  B_fp8[2] = B[23:16],  C_fp8[2] = C[23:16]
A_fp8[3] = A[31:24],  B_fp8[3] = B[31:24],  C_fp8[3] = C[31:24]
```

#### 乘法器复用

- 每路尾数乘法：`{1'b1, A_fp8[i][2:0]} × {1'b1, B_fp8[i][2:0]}` → 8bit 乘积
- **映射到 FP32 48bit 乘法器的低 8 位**
- 四路分时复用或并行展开（实现时采用并行独立乘法器，共享控制逻辑）

#### 指数计算

- 每路：`exp_sum[i][4:0] = A_fp8[i][6:3] + B_fp8[i][6:3] - 4'd7`
- FP32 扩展：`exp_fp32[i][7:0] = fp8_exp_sum[i][3:0] + 8'd113`

### 5.4 CSA 树复用方案

- 阶段 0 的 3-2 CSA 扩展为 **4-2 CSA**（增加一级压缩）
- 四路 FP8 乘积对齐后：
  - `csa_in[i][25:0] = {fp8_product[i][7:0], 18'b0}` + 指数对阶移位
- 最终两路输出：`sum[25:0]`、`carry[25:0]` 送入阶段 0 加法器

### 5.5 FP8 → FP32 指数扩展

- `exp_fp32[7:0] = fp8_exp_sum[3:0] + 8'd113`

## 6. 阶段 3：混合精度累加模式

### 6.1 模式 A：FP16×FP16 → FP32 累加

$$acc_{[31:0]} = acc_{[31:0]} + (A_{[15:0]} \times B_{[15:0]})_{\text{extend to FP32}}$$

- 尾数扩展：`product_mantissa[22:0] = {fp16_product[21:0], 1'b0}`
- 指数修正：`exp_out[7:0] = A[14:10] + B[14:10] - 5'd15 + 8'd112`

### 6.2 模式 B：FP8×4 → FP32 累加

$$acc_{[31:0]} = acc_{[31:0]} + \sum_{i=0}^{3} (A^{(i)}_{[7:0]} \times B^{(i)}_{[7:0]})_{\text{extend to FP32}}$$

- 每路尾数扩展：`product_mantissa[22:0] = {fp8_product[i][7:0], 15'b0}`
- 指数修正：`exp_out[7:0] = A[6:3] + B[6:3] - 4'd7 + 8'd120`

### 6.3 模块接口变更

- 新增输入：`acc_en`（累加使能）、`acc_in[31:0]`（外部累加器初值）
- 新增输出：`acc_out[31:0]`（累加结果）
- `mode[2:0]`：`100`=FP16→FP32 Acc，`101`=FP8×4→FP32 Acc

### 6.4 累加通路设计

```
累加寄存器: acc_reg[31:0]，复位值 = acc_in[31:0]

模式A数据流: fp16_product → extend_to_fp32 → fp32_adder → acc_reg
模式B数据流: fp8_product[0:3] → CSA_tree → fp32_adder → acc_reg

写回条件: acc_reg <= fp32_adder_out when (acc_en & valid_out)
```

### 6.5 精度扩展对照表

| 源格式 | 尾数位宽 | 扩展后尾数 `[22:0]` | 指数偏置修正 |
|--------|---------|-------------------|------------|
| FP16 | 10bit | `{product[21:0], 1'b0}` | `+112` |
| FP8 (E4M3) | 3bit | `{product[7:0], 15'b0}` | `+120` |

### 6.6 溢出保护

- 累加结果超出 FP32 范围时，`overflow` 信号拉高
- `acc_reg` 保持饱和值 `32'h7F800000`（正无穷）

## 7. 阶段 4：FP16 通路兼容 INT8

### 7.1 功能描述

**FP16 模式（`mode[0]=0`）：**

$$result_{[15:0]} = A_{[15:0]} \times B_{[15:0]} + C_{[15:0]}$$

**INT8 模式（`mode[2:0]=3'b011`）：**

$$result_{[15:0]} = A\_int_{[7:0]} \times B\_int_{[7:0]} + C\_int_{[15:0]}$$

### 7.2 INT8 位域定义

| 字段 | A\_int | B\_int | 乘积（有符号扩展） | C\_int |
|------|--------|--------|-----------------|--------|
| 符号位 | `[7]` | `[7]` | `[15]` | `[15]` |
| 数值位 | `[6:0]` | `[6:0]` | `[14:0]` | `[14:0]` |

### 7.3 INT8 乘法器复用方案

- FP16 乘法器输入位宽为 11bit（含隐含位）
- INT8 模式下：
  - `mul_in_a[10:0] = {A_int[7], A_int[7:0], 2'b0}`（符号扩展到 11bit）
  - `mul_in_b[10:0] = {B_int[7], B_int[7:0], 2'b0}`
- `mode[0]` 控制乘法器内部是否启用**补码符号扩展路径**

### 7.4 INT8 结果符号扩展与加法

- 乘积符号位：`prod_sign = A_int[7] ^ B_int[7]`
- 扩展：`product[15:0] = {{8{prod_sign}}, raw_product[7:0]}`
- 加法器输入：`product[15:0] + C_int[15:0]`，结果截断到 `[15:0]`

### 7.5 溢出检测

- INT8 乘积范围：`[-128×128, 127×127]` = `[-16384, 16129]`
- 加上 C 后若超出 `[-32768, 32767]` 则置 `overflow`

## 10. 时序图

### 三级流水线时序

```
Cycle:    1        2        3        4        5
         ┌────────┬────────┬────────┐
Input 0: │ Stage1 │ Stage2 │ Stage3 │→ Output 0
         └────────┴────────┴────────┘
                  ┌────────┬────────┬────────┐
Input 1:          │ Stage1 │ Stage2 │ Stage3 │→ Output 1
                  └────────┴────────┴────────┘
                           ┌────────┬────────┬────────┐
Input 2:                   │ Stage1 │ Stage2 │ Stage3 │→ Output 2
                           └────────┴────────┴────────┘
```

- **吞吐量**：每周期 1 个结果（流水线填满后）
- **延迟**：3 个时钟周期
- **所有精度模式共享同一流水线结构**
