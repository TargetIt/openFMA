# 阶段 1：支持 FP16（两路）设计文档

> **增量开发：在阶段 0 基础上新增 FP16 FMA 支持，同时完整保留 FP32 FMA 功能。**

## 1. 模块概述

本阶段在 FP32 FMA 基础上增量开发，支持 FP16 半精度浮点 FMA 运算：

$$result_{[15:0]} = A_{[15:0]} \times B_{[15:0]} + C_{[15:0]}$$

通过 `mode[1:0]` 信号选择精度模式：
- `2'b00`：FP32 FMA（阶段 0 功能）
- `2'b01`：FP16 FMA（本阶段新增）

## 2. 模块接口

### 输入端口

| 端口名 | 位宽 | 描述 |
|--------|------|------|
| `clk` | 1 | 系统时钟 |
| `rst_n` | 1 | 异步低电平复位 |
| `valid_in` | 1 | 输入数据有效信号 |
| `mode` | [1:0] | 精度模式：00=FP32，01=FP16 |
| `A` | [31:0] | 操作数 A（FP16 使用 `[15:0]`） |
| `B` | [31:0] | 操作数 B（FP16 使用 `[15:0]`） |
| `C` | [31:0] | 操作数 C（FP16 使用 `[15:0]`） |

### 输出端口

| 端口名 | 位宽 | 描述 |
|--------|------|------|
| `result` | [31:0] | 运算结果（FP16 结果在 `[15:0]`） |
| `valid_out` | 1 | 输出数据有效信号 |
| `overflow` | 1 | 上溢标志 |
| `underflow` | 1 | 下溢标志 |
| `inexact` | 1 | 非精确结果标志 |

## 3. FP16 位域定义

| 字段 | 位段 | 位宽 | 说明 |
|------|------|------|------|
| 符号位 | `[15]` | 1 bit | 0 = 正，1 = 负 |
| 指数位 | `[14:10]` | 5 bit | bias = 15 |
| 尾数位 | `[9:0]` | 10 bit | 含隐含 1（规格化数） |

## 4. 运算器复用说明

### 4.1 乘法器复用

- FP32 乘法器：`{1, A[22:0]} × {1, B[22:0]}` → 48bit 乘积
- FP16 映射：`{1, A[9:0]} × {1, B[9:0]}` → 22bit 乘积，**复用 FP32 乘法器低 22 位**
- 实现：FP16 模式下，乘法器高位输入强制为 0

### 4.2 指数加法器

- FP16 指数加法：`exp_sum[5:0] = A[14:10] + B[14:10] - 5'd15`
- 映射到 FP32 指数通路时加偏置修正：`exp_fp32[7:0] = fp16_exp_sum[4:0] + 8'd112`

### 4.3 共享运算器

- 移位器、CSA、加法器与阶段 0 **完全共享**
- `mode` 信号控制有效位宽

## 5. FP16 → FP32 中间格式扩展

| 源格式 | 字段 | 扩展后 FP32 格式 |
|--------|------|-----------------|
| FP16 尾数 | `product[21:0]` | `product_fp32[22:0] = {fp16_product[21:0], 1'b0}` |
| FP16 指数 | `exp_sum[4:0]` | `exp_fp32[7:0] = fp16_exp_sum[4:0] + 8'd112` |

## 6. 特殊值处理

| 输入条件 | 输出结果 |
|----------|----------|
| FP16 NaN（指数全 1，尾数非 0） | NaN（`16'h7E00`） |
| FP16 Inf（指数全 1，尾数全 0） | Inf（`16'h7C00` 或 `16'hFC00`） |
| FP16 零 | 零 |
| 所有 FP32 特殊值 | 同阶段 0 |

## 7. 三级流水线数据通路

### Stage 1：乘法阶段

- **FP32 模式**：同阶段 0
- **FP16 模式**：
  - 符号位：`sign = A[15] ^ B[15]`
  - 指数：`exp_sum = A[14:10] + B[14:10] - 15`，转换为 FP32：`+ 112`
  - 尾数：`{1, A[9:0]} × {1, B[9:0]}` → 22bit，映射到 48bit 乘法器低位

### Stage 2：对阶/CSA 阶段

- 使用统一的 FP32 中间格式
- FP16 乘积扩展后对接 FP32 CSA/加法器通路

### Stage 3：规格化/舍入阶段

- FP32 模式：同阶段 0
- FP16 模式：最终结果截取为 16bit FP16 格式

## 8. 模块层次

```
fma_fp16_stage1
├── 格式解析前端（mode 分发）
│   ├── FP32 字段提取
│   └── FP16 字段提取 + FP32 扩展
├── Stage 1 流水线寄存器
│   ├── 48bit 乘法器（共享）
│   └── 指数加法器（共享）
├── Stage 2 流水线寄存器
│   ├── 50bit 移位器（共享）
│   └── 3-2 CSA（共享）
├── Stage 3 流水线寄存器
│   ├── 50bit 加法器（共享）
│   ├── LZD / 规格化器（共享）
│   └── 舍入逻辑
└── 输出组装（mode 选择 FP32/FP16 格式）
```
