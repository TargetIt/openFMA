# FMA 单元设计提示词（多阶段渐进实现）

## 背景与目标

设计一个多精度 FMA（Fused Multiply-Add）运算单元，分阶段迭代实现，**各阶段尽量复用已有运算器**（乘法器、移位器、加法器、CSA 等），避免重复建设。

每个阶段在完成功能实现的同时，需输出一份**独立的模块设计文档和设计代码**，包含：接口定义、内部数据通路、关键时序、复用说明。

---

## 阶段 0：基础 FP32 FMA（基准实现）

### 公式

$$result_{[31:0]} = A_{[31:0]} \times B_{[31:0]} + C_{[31:0]}$$

### 位域定义

| 字段 | A / B（乘数） | C（加数） | Result |
|------|-------------|---------|--------|
| 符号位 | `[31]` | `[31]` | `[31]` |
| 指数位 | `[30:23]`（8bit，bias=127） | `[30:23]` | `[30:23]` |
| 尾数位 | `[22:0]`（23bit） | `[22:0]` | `[22:0]` |

### 内部计算

- 尾数乘法：`{1, A[22:0]} × {1, B[22:0]}` → 48bit 乘积
- 指数相加：`A[30:23] + B[30:23] - 127`（bias 修正）
- 对阶移位：将乘积与 C 尾数对齐后，经 CSA + 加法器完成累加
- 三级流水线：Stage1 = 乘法，Stage2 = 对阶/CSA，Stage3 = 规格化/舍入

### 独立设计文档要求

> **请为阶段 0 单独输出一份设计文档，包含以下内容：**

1. **模块接口**
   - 输入：`A[31:0]`、`B[31:0]`、`C[31:0]`、`clk`、`rst_n`、`valid_in`
   - 输出：`result[31:0]`、`valid_out`、`overflow`、`underflow`、`inexact`

2. **三级流水线数据通路**
   - Stage 1：符号位异或 `sign = A[31] ^ B[31]`；指数相加 `exp_sum[8:0] = A[30:23] + B[30:23] - 8'd127`；尾数乘法 `{1,A[22:0]} × {1,B[22:0]}` → `product[47:0]`
   - Stage 2：计算对阶偏移量 `shift_amt = exp_sum - C[30:23]`；移位器对齐 C 尾数；CSA 压缩乘积高位与 C 对齐尾数 → `sum[49:0]`、`carry[49:0]`
   - Stage 3：加法器 `sum + carry` → 规格化 → 舍入（round to nearest even）→ `result[31:0]`

3. **关键运算器清单**
   - 48bit 乘法器 × 1
   - 50bit 移位器 × 1
   - 3-2 CSA × 1（50bit）
   - 50bit 加法器 × 1
   - 规格化/前导零检测器 × 1

4. **特殊值处理**：NaN、Inf、零、非规格数（denormal）

## 阶段 1：支持 FP16（两路）

### 公式

$$result_{[15:0]} = A_{[15:0]} \times B_{[15:0]} + C_{[15:0]}$$

### 位域定义

| 字段 | A / B（乘数） | C（加数） | Result |
|------|-------------|---------|--------|
| 符号位 | `[15]` | `[15]` | `[15]` |
| 指数位 | `[14:10]`（5bit，bias=15） | `[14:10]` | `[14:10]` |
| 尾数位 | `[9:0]`（10bit） | `[9:0]` | `[9:0]` |

### 内部计算

- 尾数乘法：`{1, A[9:0]} × {1, B[9:0]}` → 22bit 乘积，**复用 FP32 乘法器低 22 位**
- 指数相加：`A[14:10] + B[14:10] - 5'd15`
- 乘积扩展到 FP32 中间格式后，对接阶段 0 的 CSA 和加法器通路

### 独立设计文档要求

> **请为阶段 1 单独输出一份设计文档，包含以下内容：**

1. **模块接口**
   - 新增输入：`mode[1:0]`（00=FP32，01=FP16）
   - 输入：`A[31:0]`（FP16 使用 `[15:0]`）、`B[31:0]`、`C[31:0]`
   - 输出：`result[31:0]`（FP16 结果在 `[15:0]`）

2. **复用说明**
   - FP16 尾数乘法映射到 FP32 乘法器的 `[21:0]` 位段，高位强制为 0
   - FP16 指数加法器：`exp_sum[5:0] = A[14:10] + B[14:10] - 5'd15`，结果映射回 FP32 指数通路时加偏置修正 `+112`
   - 移位器、CSA、加法器与阶段 0 完全共享，`mode` 信号控制有效位宽

3. **FP16 → FP32 中间格式扩展**
   - 尾数：`product_fp32[22:0] = {fp16_product[21:0], 1'b0}`
   - 指数：`exp_fp32[7:0] = fp16_exp_sum[4:0] + 8'd112`

4. **新增特殊值处理**：FP16 的 NaN（指数全 1，尾数非 0）、Inf（指数全 1，尾数全 0）

## 阶段 2：支持 FP8（四路）

### 公式（四路并行，以第 i 路为例）

$$result^{(i)}_{[7:0]} = A^{(i)}_{[7:0]} \times B^{(i)}_{[7:0]} + C^{(i)}_{[7:0]}, \quad i \in \{0,1,2,3\}$$

### 位域定义（E4M3 格式）

| 字段 | A / B（乘数） | C（加数） | Result |
|------|-------------|---------|--------|
| 符号位 | `[7]` | `[7]` | `[7]` |
| 指数位 | `[6:3]`（4bit，bias=7） | `[6:3]` | `[6:3]` |
| 尾数位 | `[2:0]`（3bit） | `[2:0]` | `[2:0]` |

### 内部计算

- 尾数乘法：`{1, A[2:0]} × {1, B[2:0]}` → 8bit 乘积，**复用 FP32 乘法器低 8 位**
- 指数相加：`A[6:3] + B[6:3] - 4'd7`
- 四路乘积通过**共享 CSA 树**规约，复用阶段 0 的 CSA 结构

### 独立设计文档要求

> **请为阶段 2 单独输出一份设计文档，包含以下内容：**

1. **模块接口**
   - `mode[1:0]` 扩展：00=FP32，01=FP16，10=FP8×4
   - 输入：`A[31:0]`（四路 FP8 打包为 `A[7:0]`、`A[15:8]`、`A[23:16]`、`A[31:24]`）
   - 输出：`result[31:0]`（四路 FP8 结果同样打包）

2. **四路 FP8 数据通路**
   - 解包：`A_fp8[i][7:0] = A[8*i+7 : 8*i]`，`i ∈ {0,1,2,3}`
   - 每路尾数乘法映射到 FP32 乘法器 `[7:0]` 位段
   - 四路乘积 `product[i][7:0]` 扩展到 26bit 中间格式后送入共享 CSA 树

3. **CSA 树复用方案**
   - 阶段 0 的 3-2 CSA 扩展为 4-2 CSA（增加一级压缩）
   - 四路 FP8 乘积对齐后：`csa_in[i][25:0] = {fp8_product[i][7:0], 18'b0}` + 指数对阶移位
   - 最终两路输出 `sum[25:0]`、`carry[25:0]` 送入阶段 0 加法器

4. **FP8 → FP32 指数扩展**：`exp_fp32[7:0] = fp8_exp_sum[3:0] + 8'd113`

## 阶段 3：混合精度累加模式（Accumulate 模式）

### 模式 A：两路 FP16 × FP16，累加到 FP32

$$acc_{[31:0]} = acc_{[31:0]} + \left( A_{[15:0]} \times B_{[15:0]} \right)_{\text{extend to FP32}}$$

- 尾数扩展：`product\_mantissa[22:0] = \{fp16\_product[21:0],\ 1'b0\}`
- 指数修正：`exp_out[30:23] = A[14:10] + B[14:10] - 5'd15 + 8'd112` = `A[14:10] + B[14:10] + 8'd97`
- FP32 累加器 `acc[31:0]` 经阶段 0 加法器完成累加

### 模式 B：四路 FP8 × FP8，累加到 FP32

$$acc_{[31:0]} = acc_{[31:0]} + \sum_{i=0}^{3} \left( A^{(i)}_{[7:0]} \times B^{(i)}_{[7:0]} \right)_{\text{extend to FP32}}$$

- 每路 FP8 乘积尾数扩展：`product_mantissa[22:0] = {fp8_product[i][7:0], 15'b0}`
- 指数修正：`exp_out[30:23] = A[6:3] + B[6:3] - 4'd7 + 8'd120` = `A[6:3] + B[6:3] + 8'd113`
- 四路 FP32 扩展结果经**共享 CSA 树**规约后，送入 FP32 加法器累加

### 独立设计文档要求

> **请为阶段 3 单独输出一份设计文档，包含以下内容：**

1. **模块接口**
   - 新增输入：`acc_en`（累加使能）、`acc_in[31:0]`（外部累加器初值）
   - 新增输出：`acc_out[31:0]`（累加结果）
   - `mode[2:0]` 扩展：100=FP16×FP16→FP32 acc，101=FP8×4→FP32 acc

2. **累加通路设计**
   - 累加寄存器：`acc_reg[31:0]`，复位值 = `acc_in[31:0]`
   - 模式 A 数据流：`fp16_product → extend_to_fp32 → fp32_adder → acc_reg`
   - 模式 B 数据流：`fp8_product[0:3] → CSA_tree → fp32_adder → acc_reg`
   - 写回：`acc_reg <= fp32_adder_out` when `acc_en & valid_out`

3. **精度扩展对照表**

   | 源格式 | 尾数位宽 | 扩展后尾数 `[22:0]` | 指数偏置修正 |
   |--------|---------|-------------------|------------|
   | FP16 | 10bit | `{product[21:0], 1'b0}` | `+112` |
   | FP8 (E4M3) | 3bit | `{product[7:0], 15'b0}` | `+120` |

4. **溢出保护**：累加结果超出 FP32 范围时，`overflow` 信号拉高，`acc_reg` 保持饱和值 `32'h7F800000`

## 阶段 4：FP16 通路兼容 INT8

### 公式（模式选择信号 `mode[0]`：0 = FP16，1 = INT8）

**FP16 模式（`mode[0] = 0`）：**

$$result_{[15:0]} = A_{[15:0]} \times B_{[15:0]} + C_{[15:0]}$$

**INT8 模式（`mode[0] = 1`）：**

$$result_{[15:0]} = A\_int_{[7:0]} \times B\_int_{[7:0]} + C\_int_{[15:0]}$$

### INT8 位域定义

| 字段 | A\_int | B\_int | 乘积（有符号扩展） | C\_int |
|------|--------|--------|-----------------|--------|
| 符号位 | `[7]` | `[7]` | `[15]` | `[15]` |
| 数值位 | `[6:0]` | `[6:0]` | `[14:0]` | `[14:0]` |

### 内部计算

- INT8 乘法：`A_int[7:0] × B_int[7:0]` → 16bit 有符号乘积，**复用 FP16 乘法器，`mode[0]` 控制符号扩展**
- 符号扩展：`product[15:0] = {{8{A_int[7] ^ B_int[7]}}, raw_product[7:0]}`
- 加法：复用阶段 1 的 16bit 加法器，`C_int[15:0]` 直接接入

### 独立设计文档要求

> **请为阶段 4 单独输出一份设计文档，包含以下内容：**

1. **模块接口**
   - `mode[2:0]` 最终编码：

     | `mode[2:0]` | 操作模式 |
     |-------------|---------|
     | `3'b000` | FP32 FMA |
     | `3'b001` | FP16 FMA |
     | `3'b010` | FP8×4 FMA |
     | `3'b100` | FP16×FP16 → FP32 Acc |
     | `3'b101` | FP8×4 → FP32 Acc |
     | `3'b011` | INT8 MAD（本阶段新增） |

2. **INT8 乘法器复用方案**
   - FP16 乘法器输入位宽为 11bit（含隐含位），INT8 模式下：
     - `mul_in_a[10:0] = {A_int[7], A_int[7:0], 2'b0}`（符号扩展到 11bit）
     - `mul_in_b[10:0] = {B_int[7], B_int[7:0], 2'b0}`
   - `mode[0]` 控制乘法器内部是否启用**补码符号扩展路径**

3. **INT8 结果符号扩展与加法**
   - 乘积符号位：`prod_sign = A_int[7] ^ B_int[7]`
   - 扩展：`product[15:0] = {{8{prod_sign}}, raw_product[7:0]}`
   - 加法器输入：`product[15:0]` + `C_int[15:0]`，结果截断到 `[15:0]`

4. **溢出检测**：INT8 乘积范围 `[-128×128, 127×127]` = `[-16384, 16129]`，加上 C 后若超出 `[-32768, 32767]` 则置 `overflow`

## 关键设计约束

- **运算器复用优先**：乘法器、移位器、加法器、CSA 树跨精度共享，通过 `mode[2:0]` 控制信号切换位宽与格式
- **流水线对齐**：各精度模式与阶段 0 的三级流水线对齐，Stage1 = 乘法，Stage2 = 对阶/CSA，Stage3 = 规格化/舍入
- **格式解析前端**：统一解析 FP32 / FP16 / FP8 / INT8，提取符号位、指数、尾数并路由到对应通路
- **Bias 修正统一处理**：各精度 bias（127 / 15 / 7）在指数加法阶段统一修正，复用同一个指数加法器
